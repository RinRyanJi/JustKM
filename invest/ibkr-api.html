③ 認證與環境

3 個元件, 一個 gRPC 連線

IBKR API 是 socket-based，不是 REST/HTTP。所以開發前需要設定 TWS / IB Gateway + API port + 程式 三者。

1️⃣ 下載 TWS 或 IB Gateway

  • TWS — Trader Workstation, 完整 GUI。開發、debug、看圖表都用它。
  • IB Gateway — 無 GUI 版本, 適合放在雲端 VPS/容器長期運行。

2️⃣ 啟用 API Port

進入 TWS → File → Global Configuration → API → Settings, 勾選:

  • ✅ Enable ActiveX and Socket Clients
  • ✅ Download open orders on connection（連線時自動拿到未成交單）
  • 設定 Socket port：
    • Paper Trading（模擬）：7497
    • Live Trading（真實）：7496
  • 可選：勾 Allow connections from localhost only（安全建議）
  • 勾掉 "Read-Only API"（否則下不了單）

3️⃣ 安裝 ib_insync

# 官方套件（由社群設計, 比官方 ibapi 順手很多）
pip install ib_insync

# 或者是 IB 官方 C++/Java 套件的 Python 版本（低階）
pip install ibapi

# 通常推荐 ib_insync, 99% 的人選這套

⑤ 功能 / 端點全覽

IBKR TWS API
各領域代表 endpoint

TWS API 在官方有 100+ 个 function, 分為 用戶端請求 (req*) + callback 接動 (xxxUpdate*) 兩類。以下按 ib_insync 套件修飾後的語法分類。

📊 行情數據

REQ reqMktData(contract) Tick
即時報價 bid/ask/last/volume, 可訂閱 option chain、某指數、新聞。
REQ reqHistoricalData(contract, ...) 30 年
歷史 K 線 (1 sec / 5 sec / 1 min / 5 min / 15 min / 30 min / 1 hour / 1 day)。durationStr 最長可達 5 年。
REQ reqTickers(contracts) 一次性
一次性抓多個 symbol 的 snapshot, 避免訂閱堆積。
SUB reqMarketDepth(contract, numRows=5) L2
訂閱 Order Book 深度 (5/10/20/... 檔), 需註冊 L2 行情訂閱。
REQ reqRealTimeBars(contract, ...) 5秒 K
即時 5 秒 K 線, 適合 HFT 或看見即時變化。
REQ reqScannerSubscription(...) Scanner
交易所內建掃描器：Highest Gaining / Most Active / Volume Spike / 等。

📈 下單/訂單管理

POST ib.placeOrder(contract, order) 下單
送出訂單, 回傳 Trade 物件。多種 order 類型： Market / Limit / Stop / StopLimit / TrailingStop / Algo / Adaptive
GET ib.openTrades() / openOrders() 查詢
未成交訂單。
CXL ib.cancelOrder(order) 取消
取消未成交訂單。
POST ib.whatIfOrder(contract, order) 預演
「如果下單會怎樣？」預演訊息, 含手續費、保證金、潛在閡損, 不會下單成功。
REQ ib.reqAllOpenOrders() / autoOpenOrders() 全帳戶
獲取同 clientId 全帳戶未平倉單（需設定「 Share WS 訂單」）。

💼 帳戶/持倉/成交

REQ reqAccountSummary() 總值
獲取帳戶總值、保證金、可用資金等重點指標。
REQ ib.accountValues() 詳細
獲取 170+ 個帳戶細項（cash balance、股倉市值、浮盤、未實現損益…）。
REQ reqPositions() 持倉
目前持倉。
SUB reqPositionUpdates() 即時
訂閱持倉變動推送, 生或監仓警示股用。
REQ reqPnL(account) / reqPnLSingle() P&L
即時損益報表, 更新在 trade fill 之後。
REQ reqExecutions() 成交流水
歷史成交報表, 可選時間範圍。
REQ reqFundamentalData() 基本面
從 Reuters/Refinitiv 拉取公司財務報表（年/季）, 不像 Finnhub 的多元但穩。

🎯 其他進階

REQ reqNewsBulletins() / reqNewsArticle() 新聞
拉取特定新聞 (常來自 Benzinga/Reuters)。
SUB reqUserStream() + ib.news() 新聞 streaming
新聞即時推送。
REQ reqManagedAccts() + reqFamilyCodes() 多帳戶
取得財務顧問 (FA) 架構下的多個子帳戶, 可同時管理 10+ 個子帳。
EVENT ib.orderStatusEvent / execDetailsEvent / errorEvent Callback
ib_insync 重新封裝的 event loop, 都包成 nonlocal event handler。

範例 3：下單市價單 + 回饋訂單狀態

def on_order_status(trade):
    if trade.orderStatus.status == "Filled":
        print(f"✓ 成交 {trade.contract.symbol} {trade.filledQuantity} @ ${trade.avgFillPrice}")
    elif trade.orderStatus.status in ("Cancelled", "Inactive"):
        print(f"✗ {trade.contract.symbol} 訂單 {trade.orderStatus.status}")

ib.orderStatusEvent += on_order_status

# 下 100 股 AAPL 市價單
order = MarketOrder("BUY", 100)
trade = ib.placeOrder(aapl, order)

# 等成交/超時 (5 秒)
ib.sleep(5)
print(f"訂單狀態: {trade.orderStatus.status}")
print(f"已成交: {trade.filledQuantity} / {trade.order.totalQuantity}")

範例 5：動態策略 — 均線交叉下單

import pandas as pd

def get_sma_signal(symbol, fast=20, slow=50):
    contract = Stock(symbol, "SMART", "USD")
    bars = ib.reqHistoricalData(
        contract, endDateTime='',
        durationStr='120 D',
        barSizeSetting='1 day',
        whatToShow='ADJUSTED_LAST'
    )
    df = util.df(bars)
    df['fast'] = df['close'].rolling(fast).mean()
    df['slow'] = df['close'].rolling(slow).mean()
    return df.iloc[-1]['fast'] > df.iloc[-1]['slow']

def strategy():
    while True:
        price = ticker.marketPrice()
        signal = get_sma_signal("AAPL")

        position = ib.portfolio()  # 拿倉位
        has_position = any(p.contract.symbol == "AAPL" for p in position)

        if signal and not has_position:
            order = MarketOrder("BUY", 100)
            ib.placeOrder(aapl, order)
            print(f"[{ib.reqCurrentTime()}] 買 AAPL @ ${price}")
        elif not signal and has_position:
            order = MarketOrder("SELL", 100)
            ib.placeOrder(aapl, order)
            print(f"[{ib.reqCurrentTime()}] 賣 AAPL @ ${price}")

        ib.sleep(60)  # 每分鐘檢查

⑦ Paper Trading

永久免費的
「隔離模擬」帳

IBKR 提供 Paper Trading — 獨立使用者名 + 獨立帳戶 ID + 獨立使用記錄, 跟 Live 是完全隔離的。永久免費, 可以捧一直到他關公司。

怎麼開 Paper 帳

  • 點 TWS 進去就能重新創 Paper 帳戶
  • 紙上帳戶金額是真的市場仔 USD 100 萬浮動起始 (你可以改)
  • 複委託作業: 同樣使用 clientId 端口 = 7497 (Live 是 7496)

Paper vs Live 關鍵差

面向PaperLive
Socket port74977496
帳戶類型 IDDUxxxDUxxx 或 Uxxx
起始資金USD 100 萬實際入金
下單不會真成交真實金流
行情訂閱費免費, 全部打開收取 (客戶需訂)
Margin 規則使用 Reg-T使用實記
選擇權 contract 規格可能略緩標準
💡 最佳實踐：同一個 ib_insync 脚本, 加上簡單的 if PAPER flag 切換 — Paper Trading 測試完 → 同樣程式連 Live 7496 port 就上線。

⑨ 常見坑

跳坑手冊

坑 1：clientId 重複 → 連不上

# ❌ 多個腳本都用 clientId=1, 後者被推
# ✅ 為每個腳本取不同 ID
ib_策略1 = IB().connect("127.0.0.1", 7497, clientId=1)
ib_策略2 = IB().connect("127.0.0.1", 7497, clientId=2)

# 或開啟 TWS 「Allow connections from localhost only」關 → 可以重複 ID
# 但生產不建議

坑 2：TWS 不在跑 → connect 毒倵一時找不上 port

from ib_insync import IB

ib = IB()
try:
    ib.connect("127.0.0.1", 7497, clientId=1, timeout=5)
except Exception as e:
    if "Connection refused" in str(e):
        print("請啟動 TWS / IB Gateway")

坑 3：reqMktData 不返回 × 沒 ib.sleep()

# ❌ 下個一個就讀 ticket, 報價還沒進來
ticker = ib.reqMktData(aapl)
print(ticker.bid)  # nan / None

# ✅ 給它 0.5-2 秒讓報價進來
ticker = ib.reqMktData(aapl)
ib.sleep(2)
print(ticker.bid)  # 有了!

# 或用 snapshot=True 一次性的
ticker = ib.reqMktData(aapl, snapshot="True")
ib.sleep(2)

坑 4：Paper / Live 切錯 port

# 記口訣
# 7497 = Paper
# 7496 = Live
# 4002 = IB Gateway (Live)
# 4001 = IB Gateway (Paper)

# 下單前心口訣、設環境變數
import os
PORT = int(os.getenv("IBKR_PORT", "7497"))  # 預設 paper
ib.connect("127.0.0.1", PORT, clientId=1)

坑 5：同一個ticker 多個 reqMktData → 訂閱叠造成

# ib_insync 會自動去重
for _ in range(3):
    ticker = ib.reqMktData(aapl)  # 不會建 3 個訂閱

# 但取消要手動:
ib.cancelMktData(aapl)

坑 6：用 MarketOrder 卻寫 tif=GTC

Order typetime-in-force (tif) 適用
Market / MarketOnCloseDAY (GTC 無效)
LimitDAY / GTC / IOC / GTD
Stop / StopLimitDAY / GTC
TrailingStopDAY / GTC

坑 7：socket 斷線不重接 → 策略静默失誤

import time

def keep_connection(ib):
    while True:
        if not ib.isConnected():
            print("⚠️ 離線, 重連中...")
            ib.sleep(2)
            ib.connect("127.0.0.1", 7497, clientId=1)
        ib.sleep(5)

# 在背景 thread 中跑
from threading import Thread
Thread(target=keep_connection, args=(ib,), daemon=True).start()
