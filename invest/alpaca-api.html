⓪ 給網路小白的 REST 與 WebSocket 入門

在學 Alpaca API 之前，先把兩個核心網路概念搞清楚：**REST** 與 **WebSocket**。本節用生活比喻 + 對照表，讓你 5 分鐘懂。

一句話總結

類型比喻特性Alpaca 用在哪
REST 📞 打電話叫外送：你打一次、店家回應一次、對話結束 單次請求/回應、簡單、無狀態 查帳戶、下單、看持倉、看 K 線
WebSocket 📡 LINE 群組：雙方保持連線，誰有消息就 push 過去 持續連線、雙向、伺服器主動推送 即時報價、即時成交、訂單更新推播

REST — 請問、回答、結束

REST 全名 Representational State Transfer，是現今網路最常用的 API 風格。它有 4 個核心動詞：

HTTP 動詞意思生活例子Alpaca 例子
GET查（不修改資料）看菜單查帳戶餘額 GET /v2/account
POST新增一筆下單點餐下單 POST /v2/orders
PATCH修改部分改訂單數量改單 PATCH /v2/orders/{id}
DELETE刪除取消訂單刪單 DELETE /v2/orders/{id}

每次 REST 請求都是 獨立 的，伺服器不會記得你上次問過什麼。

🔍 REST 流程（你 → 伺服器 → 你）

你（瀏覽器/程式）             Alpaca 伺服器
       │                            │
       │  ── GET /v2/account ───►   │  (1) 你發出請求
       │                            │  (2) 伺服器查詢資料庫
       │   ◄── JSON 資料 ──────    │  (3) 回傳結果
       │                            │
       │  ← 連線關閉 →               │  (4) 結束
       │                            │

💻 實際範例（用 curl 問 Alpaca 帳戶）

bash 
curl -X GET \
  -H "APCA-API-KEY-ID: YOUR_KEY_ID" \
  -H "APCA-API-SECRET-KEY: YOUR_SECRET" \
  https://paper-api.alpaca.markets/v2/account

伺服器會回一段 JSON：

{
  "id": "904837e3-3b76-47ec-b432-046db621571b",
  "account_number": "PA1234567890",
  "status": "ACTIVE",
  "cash": "10000.00",
  "portfolio_value": "15000.00",
  "buying_power": "20000.00",
  ...
}

📬 HTTP Status Code（伺服器告訴你結果的編號）

Code意思你該怎麼做
200✅ 成功拿資料用
201✅ 建立成功（POST 用）拿資料用
400❌ 你送出的格式錯了檢查 JSON、必填欄位
401❌ 沒帶 API Key檢查 Header
403❌ Key 錯或權限不足檢查 Key、餘額、PDT
404❌ 路徑錯了檢查 URL 拼字
422❌ 內容格式對但邏輯錯看 error message
429❌ 請求太頻繁睡一下再試（rate limit）
500💥 伺服器當機稍後再試或聯絡客服

WebSocket — 一直保持連線，誰有消息誰就 push

WebSocket 是瀏覽器與伺服器之間的長連線。連線建立後，雙方隨時可以互相傳訊息，不用每次都重新「打招呼」。

🔍 WebSocket 流程

你（程式）                    Alpaca 伺服器
       │                            │
       │  ── 請求升級成 WS ──►      │  (1) 用 HTTP 發起升級請求
       │   ◄── 101 Switching ───    │  (2) 伺服器同意升級
       │                            │
       │  ← 連線保持 →               │  (3) 持續連線
       │                            │
       │  ── {"action":"auth"} ►    │  (4) 你送認證
       │  ── {"action":"subscribe"} ►│  (5) 你訂閱 BTC/USD 報價
       │                            │
       │   ◄── {報價1} ──────      │  (6) 伺服器主動推播
       │   ◄── {報價2} ──────      │  (7) 又一筆成交
       │   ◄── {報價3} ──────      │  (8) 又有新報價
       │   ◄── {報價4} ──────      │
       │                            │
       │  ← 連線保持（直到你關） →    │

🎯 什麼時候該用 WebSocket？

  • ✅ 你需要即時資料（每秒數十筆的報價）
  • ✅ 你不想一直問伺服器「有新的嗎？」
  • ✅ 你想讓伺服器主動通知你（訂單成交、價格觸發）

反過來說，如果你只是想 5 秒鐘查一次 股價，REST 輪詢（每 5 秒送一次 GET）就夠了，WebSocket 反而太複雜。

📊 REST vs WebSocket 對比

面向RESTWebSocket
連線模式短連線（問完就斷）長連線（一直保持）
發起方永遠是客戶端雙方都可以
通訊單位HTTP 請求/回應JSON 訊息（雙向）
即時性差（要輪詢）極佳（伺服器主動推）
資源消耗低（短連線）較高（持續佔用 socket）
複雜度簡單（curl 就能用）較複雜（需管理連線）
適合場景CRUD、報表、定期抓資料即時報價、聊天室、通知
Alpaca 應用下單、查持倉、看 K 線即時報價、訂單狀態推播

🔧 動手試：兩個都能跑的最簡範例

以下兩個 Python 範例完全不需要 Alpaca Key，用公開測試站即可跑通。

範例 A：REST（用 httpbin.org 模擬伺服器）

python 
import requests   # pip install requests

# GET 請求 — 像打電話問「現在幾點」
r = requests.get('https://httpbin.org/get')
print(f"狀態碼: {r.status_code}")   # 200
print(f"回應 JSON: {r.json()}")

# POST 請求 — 像打電話說「我要點一份炒飯」
r = requests.post('https://httpbin.org/post',
                  json={'symbol': 'BTC/USD', 'qty': 0.01})
print(f"伺服器說: {r.json()['json']}")   # 把你送的資料 echo 回來

範例 B：WebSocket（用 echo.websocket.org 測試）

python 
import websocket   # pip install websocket-client

# 建立連線
ws = websocket.create_connection('wss://echo.websocket.org')

# 你送一句話
ws.send('Hello, 伺服器!')

# 伺服器 echo 回來
reply = ws.recv()
print(f"伺服器說: {reply}")

# 關閉
ws.close()

這個範例中：你連上一個 echo 伺服器，送什麼它就回什麼。真實 Alpaca WebSocket 差別只在伺服器會主動推播行情，不只回應你的訊息。

💡 簡單記法：
  • REST = 「我去問，伺服器答，結束」→ 適合查資料、下單
  • WebSocket = 「我們保持通話，誰有消息就 push」→ 適合即時報價
📚 延伸閱讀： MDN HTTP 動詞 · MDN WebSocket API · 維基百科 REST

① 認證與環境

所有私有 API 都需要 HTTP Header 帶上兩組 API Key。在 app.alpaca.markets 開發者後台申請。

Endpoint 網域

環境REST Base URLWebSocket
Live（實盤）https://api.alpaca.marketswss://api.alpaca.markets/stream
Paper（模擬）https://paper-api.alpaca.marketswss://paper-api.alpaca.markets/stream
Market Datahttps://data.alpaca.markets/v2wss://stream.data.alpaca.markets/v2/{iex|sip}
⚠️ Live / Paper 一定要分開：兩者用不同的 API Key 與不同網域。把測試腳本跑錯網域會直接用真錢下單。

認證範例（curl）

bash 
curl -X GET \
  -H "APCA-API-KEY-ID: {YOUR_API_KEY_ID}" \
  -H "APCA-API-SECRET-KEY: {YOUR_API_SECRET_KEY}" \
  https://paper-api.alpaca.markets/v2/account

Python SDK 認證

python 
import alpaca_trade_api as tradeapi

api = tradeapi.REST(
    key_id='YOUR_KEY_ID',
    secret_key='YOUR_SECRET_KEY',
    base_url='https://paper-api.alpaca.markets',  # 改為 api.alpaca.markets 即可上實盤
    api_version='v2'
)

account = api.get_account()
print(f"帳戶狀態: {account.status}, 可用資金: {account.cash} USD")

通用規則

  • 時間格式：所有時間使用 ISO8601 / RFC3339（含時區，例如 2026-07-11T13:30:00Z）
  • 數字精度：API 將小數以 string 回傳以保留精度，送出時建議也用 string 避免截斷
  • ID 格式：使用 UUID v4（帶連字號）。範例：904837e3-3b76-47ec-b432-046db621571b
  • Symbol 形式：可用 AAPL 或 AAPL:NASDAQ 或 AAPL:NASDAQ:us_equity 或 asset_id。大小寫敏感。
  • Rate Limit：每分鐘 200 次 / 每 API Key（paper / live 各自獨立）。超過會回 429

③ 帳戶資訊 /v2/account

GET /v2/account — 取得帳戶資訊

python 
account = api.get_account()
print(account.cash)              # 可用現金
print(account.buying_power)      # 購買力（含融資）
print(account.portfolio_value)   # 投資組合總值
print(account.equity)            # 含未實現損益的權益
print(account.daytrade_count)    # 5 個工作天內的日內交易次數（PDT 判斷）
print(account.pattern_day_trader) # 是否被標記 PDT
print(account.trading_blocked)   # 是否被禁止交易
print(account.account_blocked)   # 是否被禁止轉帳

回傳重點欄位

欄位說明
idUUID v4 帳戶 ID
account_number人類可讀的帳號
status帳戶狀態（見下表）
currency幣別（預設 USD）
cash可用現金
buying_power總購買力（含融資槓桿）
equity總權益（cash + market value）
portfolio_value持倉市值
pattern_day_traderPDT 標記（true/false）
trading_blockedtrue=僅禁止下單
account_blockedtrue=禁止下單與轉帳
shorting_enabled是否開啟放空
long_market_value做多部位市值
short_market_value放空部位市值（負值）

Account Status 狀態碼

  • ONBOARDING / SUBMISSION_FAILED / SUBMITTED — 開戶流程中
  • ACCOUNT_UPDATED — 個資更新審核中，期間暫停交易
  • APPROVAL_PENDING — 最後審批中
  • ACTIVE — ✅ 正常可交易
  • REJECTED — 申請被拒
📌 PDT 規則提醒：5 個工作天內做滿 4 次日內交易會被標記 PDT（Pattern Day Trader），帳戶權益需 ≥ $25,000 USD 才能繼續當沖。否則會被凍結帳戶 90 天。

④ 下單 Orders /v2/orders

POST /v2/orders — 下單

python 
# 市價買 1 股 AAPL
api.submit_order(
    symbol='AAPL',
    qty=1,
    side='buy',
    type='market',
    time_in_force='gtc'
)

# 限價單：$180 以下才買
api.submit_order(
    symbol='AAPL',
    qty=10,
    side='buy',
    type='limit',
    time_in_force='day',
    limit_price=180.00
)

# 止損單：股價跌破 $150 賣出
api.submit_order(
    symbol='AAPL',
    qty=10,
    side='sell',
    type='stop',
    time_in_force='gtc',
    stop_price=150.00
)

# 止損限價單（stop-limit）：跌破 $150 後只在 $148 限價賣出
api.submit_order(
    symbol='AAPL',
    qty=10,
    side='sell',
    type='stop_limit',
    time_in_force='gtc',
    stop_price=150.00,
    limit_price=148.00
)

# 追蹤止損（trailing stop）：股價回落 5% 賣出
api.submit_order(
    symbol='AAPL',
    qty=10,
    side='sell',
    type='trailing_stop',
    time_in_force='gtc',
    trail_percent=5.0   # 或用 trail_price=2.50 表示美元金額
)

關鍵參數

參數必填說明
symbol✅標的代號（支援 BTC/USD、AAPL）
qty 或 notional✅（二選一）qty 為股數；notional 為美元金額（適用 fractional）
side✅buy / sell
type✅market / limit / stop / stop_limit / trailing_stop
time_in_force✅day / gtc / opg / cls / ioc / fok
limit_price看 typelimit / stop_limit 必填
stop_price看 typestop / stop_limit 必填
trail_price / trail_percenttrailing_stop 必填追蹤金額 / 百分比（二選一）
client_order_id—自訂 ID（≤ 48 字元），便於對帳
extended_hours—true 允許盤前盤後（限價單才有效）
order_class—oco、oto、bracket（見下表）

Time In Force

  • day — 當日有效，未成交自動取消
  • gtc — 一直有效直到成交或手動取消（最多 90 天）
  • opg — 開盤單（opening cross only）
  • cls — 收盤單（closing cross only）
  • ioc — Immediate-or-Cancel，立即成交否則取消
  • fok — Fill-or-Kill，全部成交否則取消

OCO / Bracket — 多條件單

python 
# Bracket：主單 + 停損 + 停利
api.submit_order(
    symbol='AAPL',
    qty=10,
    side='buy',
    type='limit',
    time_in_force='gtc',
    limit_price=180.00,
    order_class='bracket',
    take_profit={'limit_price': 195.00},
    stop_loss={'stop_price': 175.00}
)

# OCO：停損與停利任一觸發即取消另一個
api.submit_order(
    symbol='AAPL',
    qty=10,
    side='sell',
    type='limit',
    time_in_force='gtc',
    limit_price=195.00,
    order_class='oco',
    stop_loss={'stop_price': 175.00}
)

DELETE /v2/orders/{order_id} — 取消訂單

python

# 一次取消全部

PATCH /v2/orders/{order_id} — 修改訂單

python 
api.replace_order(
    order_id='order-uuid',
    qty=20,                # 改數量
    limit_price=185.00     # 改價格
)

Order 主要狀態

  • new — 已送出，尚未到交易所
  • pending_new / accepted — 處理中
  • filled — 全部成交
  • partially_filled — 部分成交
  • canceled / expired — 已取消 / 過期
  • rejected — 被拒絕（看 rejected_message 找原因）
  • done_for_day — 今日收盤，剩餘明天繼續
💡 500 筆上限：單次 GET /v2/orders 最多回 500 筆。需要全量歷史請參考 Get All Orders recipe（用 until + after 分頁）。

⑦ 市場時鐘 Clock /v2/clock

GET /v2/clock — 查市場開收盤

python 
clock = api.get_clock()
print(f"現在 {clock.timestamp}")
print(f"市場{'開' if clock.is_open else '關'}")
print(f"下次開盤: {clock.next_open}")
print(f"下次收盤: {clock.next_close}")

# 判斷現在是否開盤（注意假日會回傳 next_open 不是當天）
if clock.is_open:
    print("可下單")
else:
    delta = clock.next_open - clock.timestamp
    print(f"離開盤還有 {delta}")

回傳欄位

欄位說明
timestamp當前時間戳（ISO 8601）
is_open是否開盤中
next_open下次開盤時間
next_close下次收盤時間

⑧ 交易日曆 Calendar /v2/calendar

GET /v2/calendar — 查交易日

⑭ WebSocket 即時串流

Alpaca 支援 [RFC6455](https://datatracker.ietf.org/doc/html/rfc6455) WebSocket， 提供 trade_updates 事件流（訂單、成交、帳戶更新）。 也支援 JSON 與 MessagePack 兩種格式。

連線 URL

  • Paper: wss://paper-api.alpaca.markets/stream
  • Live: wss://api.alpaca.markets/stream
  • Market Data: wss://stream.data.alpaca.markets/v2/{iex|sip}

連線 + 認證 + 訂閱範例

python 
import websocket, json
from threading import Thread

def on_open(ws):
    # 認證
    ws.send(json.dumps({
        'action': 'authenticate',
        'data': {
            'key_id': 'YOUR_KEY_ID',
            'secret_key': 'YOUR_SECRET_KEY'
        }
    }))
    # 訂閱 trade_updates
    ws.send(json.dumps({
        'action': 'listen',
        'data': {'streams': ['trade_updates']}
    }))

def on_message(ws, message):
    msg = json.loads(message)
    if msg['stream'] == 'trade_updates':
        event = msg['data']['event']
        order = msg['data']['order']
        print(f"[{event}] {order['symbol']} {order['side']} {order['qty']} @ {order.get('filled_avg_price')}")

def on_error(ws, err):
    print("Error:", err)

ws = websocket.WebSocketApp(
    'wss://paper-api.alpaca.markets/stream',
    on_open=on_open,
    on_message=on_message,
    on_error=on_error
)
ws.run_forever()

主要事件類型

Event說明
new訂單已送至交易所
fill全部成交（含 price、position_qty、timestamp）
partial_fill部分成交
canceled取消完成
expiredTIF 到期
done_for_day今日收盤
replaced修改完成
rejected被拒絕（看 rejected_message）
💡 推薦用法：與其 poll GET /v2/orders，不如訂閱 WebSocket — 即時、無 API 額度消耗、訂單狀態主動推播。

範例 1 — 簡易網格策略

python 
"""
網格策略：在 $170 ~ $200 間每 $5 掛一檔限價單
啟動前先取消該標的所有未成交訂單
"""
import alpaca_trade_api as tradeapi

api = tradeapi.REST(
    'KEY', 'SECRET',
    base_url='https://paper-api.alpaca.markets'
)

SYMBOL = 'AAPL'
LOW, HIGH, STEP = 170, 200, 5
QTY = 1

# 取消舊單
api.cancel_all_orders()

for price in range(LOW, HIGH, STEP):
    api.submit_order(
        symbol=SYMBOL,
        qty=QTY,
        side='buy',
        type='limit',
        time_in_force='gtc',
        limit_price=float(price)
    )
print("已掛買單網格")

範例 2 — 移動停損（trailing stop）

python 
"""
持有 AAPL 期間，每天收盤檢查，若當日高點比持倉成本高 5%，
則啟動 3% trailing stop 保護利潤
"""
import alpaca_trade_api as tradeapi
api = tradeapi.REST('K', 'S', base_url='https://paper-api.alpaca.markets')

position = api.get_position('AAPL')
entry = float(position.avg_entry_price)

# 取得今天到目前的最高價
bars = api.get_bars('AAPL', tradeapi.TimeFrame.Day, limit=1).df
today_high = float(bars['high'].iloc[-1])

# 已獲利 5% → 掛 3% trailing stop
if today_high >= entry * 1.05:
    # 先取消舊的停損單
    api.cancel_all_orders()
    api.submit_order(
        symbol='AAPL',
        qty=position.qty,
        side='sell',
        type='trailing_stop',
        time_in_force='gtc',
        trail_percent=3.0
    )
    print(f"已掛 3% trailing stop (entry ${entry}, today high ${today_high})")

"""
監控 AAPL 即時報價，跌破 $150 時市價賣出 1 股
"""
from alpaca_trade_api import Stream, REST

trade_api = REST('K', 'S', base_url='https://paper-api.alpaca.markets')

triggered = False

async def on_quote(q):
    global triggered
    if q.symbol == 'AAPL' and q.bid_price < 150 and not triggered:
        triggered = True
        trade_api.submit_order(
            symbol='AAPL', qty=1,
            side='sell', type='market',
            time_in_force='ioc'
        )
        print(f"觸發賣單 at ${q.bid_price}")

stream = Stream('K', 'S',
                base_url='wss://stream.data.alpaca.markets',
                data_feed='iex')
stream.subscribe_quotes(on_quote, 'AAPL')
stream.run()

範例 4 — PDT 保護：開盤前拒絕新單

python 
"""
每次下單前檢查是否會觸發 PDT（避免被凍結 90 天）
"""
import alpaca_trade_api as tradeapi

api = tradeapi.REST('K', 'S', base_url='https://paper-api.alpaca.markets')

def safe_submit_order(**kwargs):
    account = api.get_account()
    is_daytrade = (kwargs.get('time_in_force', 'day') == 'day')

    if account.pattern_day_trader and account.daytrade_count >= 3 and is_daytrade:
        raise Exception("⚠️ 已 PDT 標記且 5 天內做滿 4 次當沖,跳過新單避免凍結")

    if not account.pattern_day_trader and float(account.equity) < 25000 and is_daytrade:
        if account.daytrade_count >= 3:
            raise Exception("⚠️ 帳戶 < $25k 且即將觸發 PDT,跳過")

    return api.submit_order(**kwargs)

# 改用 safe_submit_order 取代直接 api.submit_order
safe_submit_order(symbol='AAPL', qty=10, side='buy', type='market', time_in_force='day')

    total_pl += pl

# 當前持倉
print("\n=== 持倉 ===")
for p in api.list_positions():

          f" | unrealized ${float(p.unrealized_pl):>8.2f} ({float(p.unrealized_plpc)*100:.2f}%)")

⑰ 訂單與部位規則速查

常見 403 原因與解法

狀況HTTP說明
帳號被禁放空卻下空單403需要 shorting_enabled=true
Easy-to-borrow 不足403改找 easy_to_borrow=true 標的
購買力不足403檢查 buying_power 是否涵蓋
非融資標的用融資買403篩 marginable=true
想做多翻空直接單403目前不支援，需先平倉再反向
Rate Limit429每分鐘 200 次，加 sleep / 走 WebSocket

市價單的特殊處理

  • 做多市價單：會被轉為 marketable limit，價格 collar +2.5% ~ +4%（防止滑價過大）
  • 做空市價單：不會轉，但會做 102.5% ~ 104% 的購買力預檢
  • 可融資標的：用 buying_power 判斷可用額度
  • 不可融資標的：需用現金 cash

短部位規則

  • 放空單的 side=sell，無新增參數
  • 短部位 qty 和 market_value 都是負值
  • 目前只能放空 easy_to_borrow=true 的標的
  • 不能直接從多翻空（或空翻多），需先平倉再反向
  • 已有 pending sell 時不能再下 buy（會 403）

部位顯示規則

  • 短部位以負值表示：例如賣空 10 股 AAPL，qty=-10、market_value=-1500
  • 放空利潤顯示為負數時表示虧損
