⓪ 給網路小白的 REST 與 WebSocket 入門

在學 Alpaca API 之前，先把兩個核心網路概念搞清楚：**REST** 與 **WebSocket**。本節用生活比喻 + 對照表，讓你 5 分鐘懂。

一句話總結

類型比喻特性Alpaca 用在哪
REST 📞 打電話叫外送：你打一次、店家回應一次、對話結束 單次請求/回應、簡單、無狀態 查帳戶、下單、看持倉、看 K 線
WebSocket 📡 LINE 群組：雙方保持連線，誰有消息就 push 過去 持續連線、雙向、伺服器主動推送 即時報價、即時成交、訂單更新推播

REST — 請問、回答、結束

REST 全名 Representational State Transfer，是現今網路最常用的 API 風格。它有 4 個核心動詞：

HTTP 動詞意思生活例子Alpaca 例子
GET查（不修改資料）看菜單查帳戶餘額 GET /v2/account
POST新增一筆下單點餐下單 POST /v2/orders
PATCH修改部分改訂單數量改單 PATCH /v2/orders/{id}
DELETE刪除取消訂單刪單 DELETE /v2/orders/{id}

每次 REST 請求都是 獨立 的，伺服器不會記得你上次問過什麼。

🔍 REST 流程（你 → 伺服器 → 你）

你（瀏覽器/程式）             Alpaca 伺服器
       │                            │
       │  ── GET /v2/account ───►   │  (1) 你發出請求
       │                            │  (2) 伺服器查詢資料庫
       │   ◄── JSON 資料 ──────    │  (3) 回傳結果
       │                            │
       │  ← 連線關閉 →               │  (4) 結束
       │                            │

💻 實際範例（用 curl 問 Alpaca 帳戶）

bash 
curl -X GET \
  -H "APCA-API-KEY-ID: YOUR_KEY_ID" \
  -H "APCA-API-SECRET-KEY: YOUR_SECRET" \
  https://paper-api.alpaca.markets/v2/account

伺服器會回一段 JSON：

{
  "id": "904837e3-3b76-47ec-b432-046db621571b",
  "account_number": "PA1234567890",
  "status": "ACTIVE",
  "cash": "10000.00",
  "portfolio_value": "15000.00",
  "buying_power": "20000.00",
  ...
}

📬 HTTP Status Code（伺服器告訴你結果的編號）

Code意思你該怎麼做
200✅ 成功拿資料用
201✅ 建立成功（POST 用）拿資料用
400❌ 你送出的格式錯了檢查 JSON、必填欄位
401❌ 沒帶 API Key檢查 Header
403❌ Key 錯或權限不足檢查 Key、餘額、PDT
404❌ 路徑錯了檢查 URL 拼字
422❌ 內容格式對但邏輯錯看 error message
429❌ 請求太頻繁睡一下再試（rate limit）
500💥 伺服器當機稍後再試或聯絡客服

WebSocket — 一直保持連線，誰有消息誰就 push

WebSocket 是瀏覽器與伺服器之間的長連線。連線建立後，雙方隨時可以互相傳訊息，不用每次都重新「打招呼」。

🔍 WebSocket 流程

你（程式）                    Alpaca 伺服器
       │                            │
       │  ── 請求升級成 WS ──►      │  (1) 用 HTTP 發起升級請求
       │   ◄── 101 Switching ───    │  (2) 伺服器同意升級
       │                            │
       │  ← 連線保持 →               │  (3) 持續連線
       │                            │
       │  ── {"action":"auth"} ►    │  (4) 你送認證
       │  ── {"action":"subscribe"} ►│  (5) 你訂閱 BTC/USD 報價
       │                            │
       │   ◄── {報價1} ──────      │  (6) 伺服器主動推播
       │   ◄── {報價2} ──────      │  (7) 又一筆成交
       │   ◄── {報價3} ──────      │  (8) 又有新報價
       │   ◄── {報價4} ──────      │
       │                            │
       │  ← 連線保持（直到你關） →    │

🎯 什麼時候該用 WebSocket？

  • ✅ 你需要即時資料（每秒數十筆的報價）
  • ✅ 你不想一直問伺服器「有新的嗎？」
  • ✅ 你想讓伺服器主動通知你（訂單成交、價格觸發）

反過來說，如果你只是想 5 秒鐘查一次 股價，REST 輪詢（每 5 秒送一次 GET）就夠了，WebSocket 反而太複雜。

📊 REST vs WebSocket 對比

面向RESTWebSocket
連線模式短連線（問完就斷）長連線（一直保持）
發起方永遠是客戶端雙方都可以
通訊單位HTTP 請求/回應JSON 訊息（雙向）
即時性差（要輪詢）極佳（伺服器主動推）
資源消耗低（短連線）較高（持續佔用 socket）
複雜度簡單（curl 就能用）較複雜（需管理連線）
適合場景CRUD、報表、定期抓資料即時報價、聊天室、通知
Alpaca 應用下單、查持倉、看 K 線即時報價、訂單狀態推播

🔧 動手試：兩個都能跑的最簡範例

以下兩個 Python 範例完全不需要 Alpaca Key，用公開測試站即可跑通。

範例 A：REST（用 httpbin.org 模擬伺服器）

python 
import requests   # pip install requests

# GET 請求 — 像打電話問「現在幾點」
r = requests.get('https://httpbin.org/get')
print(f"狀態碼: {r.status_code}")   # 200
print(f"回應 JSON: {r.json()}")

# POST 請求 — 像打電話說「我要點一份炒飯」
r = requests.post('https://httpbin.org/post',
                  json={'symbol': 'BTC/USD', 'qty': 0.01})
print(f"伺服器說: {r.json()['json']}")   # 把你送的資料 echo 回來

範例 B：WebSocket（用 echo.websocket.org 測試）

python 
import websocket   # pip install websocket-client

# 建立連線
ws = websocket.create_connection('wss://echo.websocket.org')

# 你送一句話
ws.send('Hello, 伺服器!')

# 伺服器 echo 回來
reply = ws.recv()
print(f"伺服器說: {reply}")

# 關閉
ws.close()

這個範例中：你連上一個 echo 伺服器，送什麼它就回什麼。真實 Alpaca WebSocket 差別只在伺服器會主動推播行情，不只回應你的訊息。

💡 簡單記法：
  • REST = 「我去問，伺服器答，結束」→ 適合查資料、下單
  • WebSocket = 「我們保持通話，誰有消息就 push」→ 適合即時報價
📚 延伸閱讀： MDN HTTP 動詞 · MDN WebSocket API · 維基百科 REST

① 認證與環境

認證方式與 Stock Trading API 相同 — Header 帶 APCA-API-KEY-ID 與 APCA-API-SECRET-KEY。

Endpoint 網域

環境REST Base URLWebSocket
Live（實盤）https://api.alpaca.marketswss://api.alpaca.markets/stream
Paper（模擬）https://paper-api.alpaca.marketswss://paper-api.alpaca.markets/stream
Crypto 即時行情—wss://stream.data.alpaca.markets/v1beta2/crypto
⚠️ Crypto 也分 paper / live：同樣的 API Key 在 paper 環境開 Paper Trading 模式，在 live 環境就要開真錢模式。切換時只改 base_url。

Python SDK 連線範例

python 
import alpaca_trade_api as tradeapi

# Crypto 跟 Stock 用同一個 SDK 與同一組 Key
api = tradeapi.REST(
    key_id='YOUR_KEY_ID',
    secret_key='YOUR_SECRET_KEY',
    base_url='https://paper-api.alpaca.markets',
    api_version='v2'
)

# 帳戶查詢方式完全相同
acct = api.get_account()
print(f"現金: {acct.cash} USD")

通用規則（與 Stock API 一致）

  • 時間格式：ISO8601 / RFC3339（含時區）
  • 數字精度：小數以 string 回傳以保留精度，送出時建議也用 string
  • ID 格式：UUID v4（帶連字號）
  • Symbol 形式：BTC/USD 或 BTCUSD 或 BTC/USD:CRYPTO 或 asset_id。大小寫敏感。
  • Rate Limit：每分鐘 200 次 / 每 API Key。超過回 429

② Crypto API 概觀

Alpaca Crypto API 讓你用同一組 API Key 即可交易加密貨幣。支援：

  • 52+ 交易對 × 20+ 幣種
  • 計價單位：USD、BTC、USDT
  • 7×24 全年無休（不像股票有開收盤）
  • 支援 市價 / 限價 / 停損 / 追蹤止損 / OCO / Bracket 全部訂單類型
  • 即時行情透過 WebSocket v1beta2/crypto
  • 無 PDT 限制（不受 pattern day trader 規則影響）
  • 無融資 / 放空（Crypto 為現貨交易，不能放空）

與 Stock API 的差異速覽

項目StockCrypto
交易時間美股時段 + 盤前盤後7×24 全天候
放空✅ 可❌ 不可（僅現貨）
融資✅ 可（依帳戶）❌ 不可
PDT 限制✅ 有❌ 無
Symbol 格式AAPLBTC/USD、ETH/BTC
最小下單單位1 股（可碎股）依標的而異（常見 0.0001 BTC）
持有幣種結算USDUSD / BTC / USDT（依交易對）
即時行情 WebSocket/v2/iex 或 /v2/sip/v1beta2/crypto
ℹ️ Crypto Account 是 Stock 帳戶的延伸：你不需要再開 Crypto 帳戶，在同一個 Alpaca 帳戶下， get_account() 回傳的 cash 是 USD 餘額，positions 會同時列出股票與幣幣。 加密貨幣無法作為保證金。

④ 下單 Orders /v2/orders

POST /v2/orders — 下單範例

python 
# 市價買 $100 美元的 BTC
api.submit_order(
    symbol='BTC/USD',
    notional='100',         # 用美元金額（碎幣交易）
    side='buy',
    type='market',
    time_in_force='gtc'
)

# 限價買 0.01 BTC at $60,000
api.submit_order(
    symbol='BTC/USD',
    qty='0.01',
    side='buy',
    type='limit',
    time_in_force='gtc',
    limit_price='60000'
)

# 限價賣 ETH at $3,500（持倉需 ≥ 0.01 ETH）
api.submit_order(
    symbol='ETH/USD',
    qty='0.1',
    side='sell',
    type='limit',
    time_in_force='gtc',
    limit_price='3500'
)

# BTC/ETH 對（非 USD 計價）
api.submit_order(
    symbol='BTC/ETH',
    qty='0.01',
    side='buy',
    type='market',
    time_in_force='ioc'
)

# 追蹤止損：BTC 從高點回落 5% 賣出
api.submit_order(
    symbol='BTC/USD',
    qty='0.01',
    side='sell',
    type='trailing_stop',
    time_in_force='gtc',
    trail_percent='5.0'
)

# Bracket：BTC 買入後自動掛停損 + 停利
api.submit_order(
    symbol='BTC/USD',
    notional='1000',
    side='buy',
    type='market',
    time_in_force='gtc',
    order_class='bracket',
    take_profit={'limit_price': '70000'},
    stop_loss={'stop_price': '58000'}
)

關鍵參數

參數必填說明
symbol✅BTC/USD、ETH/BTC、USDT/USD 等
qty 或 notional✅（二選一）qty=幣數，notional=美元金額（最小通常 $1）
side✅buy / sell（僅現貨）
type✅market / limit / stop / stop_limit / trailing_stop
time_in_force✅gtc / ioc / fok（Crypto 不支援 day、opg、cls）
limit_price看 typelimit / stop_limit / bracket take_profit 必填
stop_price看 typestop / stop_limit / bracket stop_loss 必填
trail_percent / trail_pricetrailing_stop 必填百分比或美元金額
client_order_id—自訂 ID（≤ 48 字元）
⚠️ Crypto 不支援的 TIF：沒有 day、opg、cls。使用會回 400。 想限當日自動取消，請在 client 端自己用 cron / 排程刪單。

Order 主要狀態

  • new / accepted — 送出、等待撮合
  • filled — 完全成交
  • partially_filled — 部分成交（Crypto 較少見，因多為市價或限價全成）
  • canceled / expired — 已取消 / 到期
  • rejected — 拒絕（看 rejected_message 找原因）

取消與查詢（與 Stock 完全相同）

python 
# 查開放中訂單
orders = api.list_orders(status='open', symbols=['BTC/USD', 'ETH/USD'])

# 取消單筆
api.cancel_order('order-uuid')

# 取消所有
api.cancel_all_orders()

# 用 client_order_id 查

# 修改（僅 gtc 限價單可改）
api.replace_order('order-uuid', qty='0.02', limit_price='59000')

連線範例

python 
import websocket, json
from threading import Thread

def on_open(ws):
    ws.send(json.dumps({
        'action': 'auth',
        'key': 'YOUR_KEY_ID',
        'secret': 'YOUR_SECRET_KEY'
    }))
    ws.send(json.dumps({
        'action': 'subscribe',
        'trades': ['BTC/USD'],
        'quotes': ['ETH/USD'],
        'bars': ['BTC/USD', 'ETH/USD'],
        'orderbooks': ['BTC/USD']
    }))

def on_message(ws, msg):
    arr = json.loads(msg)
    for item in arr:
        t = item.get('T')
        if t == 't':    # trade
            print(f"TRADE {item['S']} px={item['p']} qty={item['s']}")
        elif t == 'q':  # quote
            print(f"QUOTE {item['S']} bid={item['bp']} ask={item['ap']}")
        elif t == 'b':  # bar
            print(f"BAR {item['S']} o={item['o']} c={item['c']} v={item['v']}")
        elif t == 'o':  # orderbook
            print(f"OB {item['S']} bids={len(item['b'])} asks={len(item['a'])}")

ws = websocket.WebSocketApp(
    'wss://stream.data.alpaca.markets/v1beta2/crypto',
    on_open=on_open,
    on_message=on_message
)
ws.run_forever()

使用 SDK（推薦）

python 
from alpaca_trade_api import Stream

stream = Stream(
    key_id='YOUR_KEY_ID',
    secret_key='YOUR_SECRET_KEY',
    base_url='wss://stream.data.alpaca.markets',
    data_feed='crypto'   # 不同於股票的 'iex'/'sip'
)

async def on_trade(t):
    print(f"{t.symbol} ${t.price} x {t.size}")

async def on_bar(b):
    print(f"{b.symbol} OHLCV: {b.open}/{b.high}/{b.low}/{b.close}/{b.volume}")

stream.subscribe_trades(on_trade, 'BTC/USD', 'ETH/USD')
stream.subscribe_bars(on_bar, 'BTC/USD')
stream.run()

405訂閱數超過方案上限
406已有其他連線（單帳戶單連線限制）
407處理太慢（buffer 滿）

範例 3 — 動態網格（BTC）

python 
"""
在當前價 ±10% 範圍內每 2% 掛一檔限價單，金額 $50
"""
import alpaca_trade_api as tradeapi
api = tradeapi.REST('K', 'S', base_url='https://paper-api.alpaca.markets')

SYMBOL = 'BTC/USD'
GRID_COUNT = 10
STEP_PCT = 0.02
AMOUNT = '50'

# 清掉舊單
api.cancel_all_orders()

# 取當前價
last = float(api.get_snapshot(SYMBOL).latest_trade.price)

orders = []
for i in range(-GRID_COUNT//2, GRID_COUNT//2 + 1):
    price = round(last * (1 + i * STEP_PCT), 2)
    if i == 0: continue
    side = 'buy' if price < last else 'sell'
    try:
        o = api.submit_order(
            symbol=SYMBOL,
            notional=AMOUNT,
            side=side,
            type='limit',
            time_in_force='gtc',
            limit_price=str(price)
        )
        orders.append(o)
        print(f"  {side} @ ${price}")
    except Exception as e:
        print(f"  Skip {price}: {e}")

print(f"\n總共掛 {len(orders)} 檔")

範例 4 — 即時價觸發（市價單）

python 
"""
監控 BTC 即時報價，跌破 $58,000 時市價賣出全部持倉
"""
from alpaca_trade_api import Stream, REST
import asyncio

trade_api = REST('K', 'S', base_url='https://paper-api.alpaca.markets')
fired = False

async def on_quote(q):
    global fired
    if q.symbol == 'BTC/USD' and q.bid_price < 58000 and not fired:
        fired = True
        btc = trade_api.get_position('BTC/USD')
        trade_api.submit_order(
            symbol='BTC/USD',
            qty=btc.qty,
            side='sell',
            type='market',
            time_in_force='ioc'
        )
        print(f"觸發賣單 bid=${q.bid_price}")
        # 結束 stream
        await asyncio.sleep(1)

stream = Stream(
    'K', 'S',
    base_url='wss://stream.data.alpaca.markets',
    data_feed='crypto'
)
stream.subscribe_quotes(on_quote, 'BTC/USD')
stream.run()

範例 5 — 即時 OHLCV 進 SQLite

python 
"""
把 BTC/USD 的 1 分 K 寫入 SQLite，便於後續 backtest
"""
import sqlite3
from alpaca_trade_api import Stream
import datetime

db = sqlite3.connect('/tmp/crypto_bars.db')
db.execute('''
CREATE TABLE IF NOT EXISTS bars (
    symbol TEXT, ts TEXT, open REAL, high REAL,
    low REAL, close REAL, volume REAL,
    PRIMARY KEY (symbol, ts)
)
''')

async def on_bar(b):
    db.execute('''
        INSERT OR REPLACE INTO bars
        VALUES (?, ?, ?, ?, ?, ?, ?)
    ''', (b.symbol, b.timestamp.isoformat(),
          b.open, b.high, b.low, b.close, b.volume))
    db.commit()
    print(f"{b.symbol} @ {b.timestamp} c={b.close}")

stream = Stream('K', 'S',
                base_url='wss://stream.data.alpaca.markets',
                data_feed='crypto')
stream.subscribe_bars(on_bar, 'BTC/USD', 'ETH/USD')
stream.run()

⑩ Crypto vs Stock 關鍵差異

下單與部位差異

  • 無放空：Crypto 僅現貨，qty 永遠為正
  • 無融資：下單前檢查 cash（USD）是否足夠
  • 無 PDT：當沖無限制，daytrade_count 對 Crypto 不適用
  • 無 fragments 美股那樣碎股限制，但有 min_trade_qty 與 min_trade_notional
  • TIF 限制：只有 gtc / ioc / fok

常見 403/422 原因

狀況HTTP解法
USD 餘額不足403先入金或賣其他幣
持倉幣數不足403賣單 qty ≤ 持倉
下單量 < min_trade_qty422提高 qty
下單金額 < min_trade_notional422提高 notional
價格非 price_increment 倍數422round 到 increment
使用 day/opg/cls TIF422改用 gtc
嘗試放空403不可放空，先建倉再賣

幣幣轉換（USDT/USD 結算）

Alpaca 的 USDT/USD 對視為 同類資產，交易 USDT/USD 不會扣你的 USD 餘額，而是用你已有的 USDT。如果你持有 USDT 而想換成 USD，可以：
